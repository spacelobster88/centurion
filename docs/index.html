Why Centurion?

Built for when 1-2 agents isn't enough.

⚔

Roman Military Hierarchy

Organize agents into Legions, Centuries, and Legionaries. Intuitive fleet management at any scale with clear chain of command.

⚙

K8s-Inspired Scheduling

Admission control, resource requests/limits, and hardware probing. The scheduler knows exactly how many agents your machine can handle.

🔊

Fleet Broadcasting

Send real-time instructions to all agents, a specific legion, or a specific century. Change agent behavior on the fly.

📈

Auto-Scaling (Optio)

Each century has an Optio that monitors queue depth and scales agents up or down in real time. No manual intervention needed.

⚡

One-Click Quickstart

centurion quickstart — probes hardware, recommends config, creates a fleet, and starts the server. One command.

🔗

5 Integration Methods

REST API, MCP Server, Claude Code Skill, Google A2A Protocol, and Python Library. Fits into any AI ecosystem.

Case Study

Real-world parallel research at scale.

COMPLETED

Researching Top 20 AI Agent Companies in Parallel

We deployed Centurion to research the top 20 AI agent companies simultaneously. Each legionary was assigned one company to analyze — covering product offerings, funding, team size, tech stack, and competitive positioning. What would take a single agent 4+ hours sequentially was completed in 12 minutes with 20 agents running in parallel.

20 Parallel Agents
12 min Total Wall Time
~1.2M Tokens Consumed
20x Speedup vs Single
$2.40 Total API Cost

Mission Timeline

# The entire operation in 5 commands centurion quickstart --agent-type claude_cli curl -X POST localhost:8100/api/centurion/legions \ -d '{"name": "Research Ops"}' curl -X POST localhost:8100/api/centurion/legions/default/centuries \ -d '{"agent_type": "claude_cli", "min_legionaries": 20, "max_legionaries": 20}' curl -X POST localhost:8100/api/centurion/legions/default/tasks \ -d '{"prompts": ["Research OpenAI", "Research Anthropic", "Research Google DeepMind", ...]}' curl -X POST localhost:8100/api/centurion/broadcast \ -d '{"message": "Include funding details", "target": "all"}'
